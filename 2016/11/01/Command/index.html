工作常用命令：Linux、eclipse、sublime

Linux

  • man
    操作说明(manual ls)后跟命令 查看该命令的操作说明

  • ls -l
    列出目录下的所有文件 -l文件详细信息(相当于ll)(d为文件夹，-为文件)list

    ls -lh  显示出文件大小

  • cd
    切换目录(/ 到根目录; ..到上层目录; - 到上次工作的目录;~ 到当前用户根目录)change directory

  • pwd
    显示当前目录print working directory
  • mkdir
    创建目录make directories
    mkdir -p folder1/folder2    连续创建两个目录
  • touch
    创建文件
  • rm -rf
    强制递归删除文件或文件夹及子文件(recursive递归 force强制)remove
  • cp -r
    复制(-r递归复制)copy
  • mv
    移动或更名现有的文件或目录(-b若覆盖文件则先行备份 -f直接覆盖)move
  • vi
    编辑文体文件 参考
    w:保存当前文件    wq保存并退出 q!:不保存并退出
  • file 1.txt
    查看文件类型，也可用来辨别一些文件的编码格式
  • diff
    比较文件的差异 diff 1.txt 2.txt -y -W 50 y并列展示 W为宽度
  • more
    查看文件，一页一页显示(空格：下一页；b：上一页；) q 退出more
  • cat
    查看文件，全部显示 cat -n 对文件进行行编号
    cat -n 1.txt | head -n 5 | tail -n 2    显示第5行到7行
  • head
    默认打印其相应文件的开头10行(-n 显示的行数)
  • tail
    打印其相应文件的尾部(-f 循环读取follow；-v 显示详细的处理信息；-n 显示行数；ctrl c 终止打印)
  • find
    查找文件(find ~ -name “.log” -print 根目录$HOME中查找文件名符合.log的文件;
    find . -name "*.log" -print 在当前目录及子目录中查找所有的"*.log"文件;
find / -name "*.log" -print 在根目录及子目录中查找所有的"*.log"文件)
  • grep
    在指定的文本文件中查找指定的字符串输出一行(Global Regular Expression Print)
    grep -c "abc" 1.txt   统计abc在1.txt中共几行
grep -n "abc" 1.txt   能显示出abc在1.txt中第几行
grep "abc" . -r       在当前多级目录中对文本进行递归(recurse)搜索 .表示当前目录
  • wc
    统计文本文件的行数、字数、字符数(wordcount)
  • clear
    清除终端屏幕
  • gzip -d
  • tar -zxvf
    解开压缩文件

  • zip
    将当前目录下的所有文件和文件夹全部压缩成myfile.zip文件,－r表示递归压缩子目录下所有文件.

    zip -r myfile.zip ./*

  • unzip
    -o:不提示的情况下覆盖文件;-d:-d /home/sunny 指明将文件解压缩到/home/sunny目录下

    unzip -o -d /home/sunny myfile.zip  把myfile.zip文件解压到 /home/sunny/
  • sz
    下载到本地
    sz filename
  • wget
    从URL中下载
    wget http://cn.wordpress.org/wordpress-3.1-zh_CN.zip

  • rz -be

    上传到服务器  -b 以二进制方式，默认为文本方式   -e 对所有控制字符转义

  • |
    管道连接符

    例：ls -l | grep "^d"   列出当前目录下的所有文件信息
ls -l | grep "^-" | wc -l   当前目录下有多少个文件

  • whereis
    查看执行命令ls的文件地址(查验是否已安装ls命令)

    whereis ls

  • wall
    通知所有人(warn all)(wall data 后跟命令的话，会将输出结果展示给所有人)

  • >
    重定向 把ls命令的打印结果写入test.txt文件中

    ls > test.txt

    把123写入1.txt中

    echo "123" > 1.txt
  • useradd
    添加用户

  • chmod

    文件权限
drwxr-xr-x 8 root  root      4096 Dec  3  2013 chenliang
drwxr-xr-x 2 root  root      4096 Oct 11  2013 cms
-rwxr-xr-x 1 root  root      1014 Aug 31  2013 ct.sh
drwxr-xr-x 3 root  root      4096 Dec 29 23:52 data
drwxr-xr-x 3 root  root      4096 Mar 27 22:09 jishu
第一字母：d为文件夹 -为文件 后面9位3组
r：read    w:write   x:execute
第一组：文件所有者
第二组：文件所有者同组人员
第三组：其他人员
修改权限：
chmod   (change modify)
例：
chmod +x 3.txt  三组人员都有执行权限
chmod -x 3.txt  三组人员都没有执行权限
chmod u+x 3.txt 第一组人员有执行权限(user)
chmod g+x 3.txt 第二组人员有执行权限(group)
chmod o+x 3.txt 第三组人员有执行权限(other)
三组权限分别用三个8进制数代表
-rwxr-xr-x 3.txt
chmod 753 3.txt
修改权限后结果为：-rwxr-x-wx
修改文件所有者
chown user2 3.txt

  • ps
    查看进程(process status)查看tomcat进程

    ps -ef | grep tomcat
  • crontab
    定时任务相关
  • scp
    服务器间文件互传

eclipse

  • Ctrl E 快速显示当前Editer的下拉列表
  • Ctrl F6 切换到下一个Editor
  • Ctrl F7 切换到下一个Perspective

  • Ctrl F8 切换到下一个View

  • Ctrl Shift+X 把当前选中的文本全部变为大写

  • Ctrl Shift+Y 把当前选中的文本全部变为小写
  • Ctrl Shift+P 定位到对于的匹配符(譬如{})

  • Ctrl Shift+R 打开调动方法记录

  • Alt Shift 数字 多窗口

  • Alt Shift R 重命名
  • Alt Shift M 抽取方法
  • Alt Shift C 修改函数结构(比较实用,有N个函数调用了这个方法,修改一次搞定)
  • Alt Shift Z 重构的后悔药(Undo)

sublime text 3

更多参考

  • Ctrl alt F 格式化代码
  • Ctrl R 打开方法结构 快速定位方法
  • Ctrl P 快速打开常用历史文件
  • Ctrl shift 1,2,4 打开多个视图
  • Ctrl Shift D 复制光标所在整行，插入在该行之前
  • Ctrl Shift K 删除整行
  • Ctrl Shift L 鼠标选中多行（按下快捷键），即可同时编辑这些行
  • Ctrl Shift M 选择括号内的内容（按住-继续选择父括号）
  • Ctrl Shift / 注释已选择内容
  • Ctrl Shift ↑ 可以移动此行代码，与上行互换
  • Ctrl Shift ↓ 可以移动此行代码，与下行互换
  • Ctrl+M 由md文件生成html预览 hexo使用

git

  • git init 初始化本地库
  • git add 1.txt 增加文件到本地库
  • git commit -m "说明" 提交到本地库并加上说明
  • git log 查看操作日志
  • git reset --hard HEAD^ 回退到上个版本
  • git reset --hard HEAD~5 回退到上5个版本
  • git reset --hard 36281 回退到36281版本 版本号可写不全
  • git reflog 查看使用过的命令记录
  • git diff HEAD -- 1.txt 查看工作区和版本库里面最新版本的区别
  • git checkout -- 1.txt 可以丢弃工作区的修改
  • git reset HEAD 1.txt 可以把暂存区的修改撤销掉(unstage) 重新放回工作区
  • git push origin master 提交到远程git
