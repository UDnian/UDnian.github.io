代码清洁之道

类名

类名和对象应该是名词或名词短语，如Customer、WikiPage、Account。避免Manager、Data、Info。类名不应当是动词。

方法名

方法名应当是动词或动词短语，如postPayment、deletePage、sava，或 依Javabean标准加上get、set、is前缀。

函数规则

函数的第一规则是短小，第二规则还是短小；
只做一件事；
每个函数一个抽象层级；

抽离Try/Catch代码块

Try/Catch代码块丑陋不堪。他们搞乱了代码结构，把错误处理与正常流程混为一谈。最好把些代码块的主体部分抽离出来，另外形成函数。


如果关键字try在某个函数中存在，它就该是这个函数的第一个单词，而且在catch/finally 代码块后面也不该有其他内容。
